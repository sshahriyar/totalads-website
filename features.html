TotalADS: Total Anomaly Detection System

TotalADS is an Eclipse-based open source host based anomaly detection system to detect anomalies on a host using software execution traces or software logs. TotalADS allows to train different anomaly detection techniques (e.g., HMM, sequence matching,etc.) on offline traces or online trace stream, detect anomalies in new traces or trace stream, and visualize traces for forensic analysis.TotalADS can be used with different of trace formats, such as CTF, XML and text formats, and new trace formats can also be easily added. TotalADS will soon be available for download in Eclipse Market place.

TotalADS is actually based on Tracing and Monitoring Framework (TMF), a tracing platform developed by Ericsson.TMF provides different views for trace inspection and TotalADS enhanced TMF by providing automated anomaly detection in traces. TotalADS, including TMF, provides following views for trace inspection and automated anomaly detection:

Modeling View

In the modeling view, an anomaly detection technique can be used to build a model of traces by training and validating itself on a set of existing traces. The modeling view also allows to adjust different settings of the anomaly detection techniques. Three anomaly detection techniques are currently supported: Hidden Markov Model, Sequence Matching and Kernel State Modeling.

Diagnosis View

This view tests an already built model on execution traces or software logs. It shows detailed results on each trace. Decision thresholds of models can also be adjusted in this view to decrease or increase the number of false alarms.

Live Monitor View

This view allows to train and test models in real time by automatically collecting LTTng kernel space traces from a Linux system. It has two modes: (a) train and test mode, and (b) test mode. In the ‘train and test’ mode, a trace is collected from a remote system over SSH, tested on a model, and then the same trace is used for training. This mode should be used in a controlled environment when it is certain that all the system executions are normal (attack-free). Several models can be trained and tested simultaneously in this mode. In the live test mode (i.e., when the system is in operation), traces from real usage of the system are collected from a remote host and the models are evaluated on them.

Events View

When a model flags a trace as anomalous, an analyst may need to analyze it in detail. An individual trace can be analyzed in detail in this view. It shows all the events and their details in the chronological order of their occurrences. For example, a system call and its arguments, occurrences of page faults in memory, scheduling events, interrupt requests, etc. can be analyzed in detail over here.

Control Flow View

This view is suitable for individual kernel space traces. It shows the states of different processes in a system, their inter process communication, and the mode of execution. For example, a process can be in wait-blocked state (shown as yellowish green bar), in a user space state (shown as a green bar), in a kernel space (shown as blue bar), can be waiting for CPU (shown as brown bar) and can be interrupted (shown as pink bar). Lines between bars shows inter process communication.

Resource View

This view is particularly useful if there are multiple CPUs in a system and it is suitable for individual kernel space traces. In this view, utilization of CPUs, interrupts and other system resources by processes can be observed with respect to time. The color coding of processes is the same as the Control Flow view.

Histogram View

In this view frequency of events of a trace are plotted on a time line. This view is useful to detect performance issues in a system. For example, if there are multiple CPUs in a system, and the amount of load suddenly increases on them at some time, then this view can be used to quickly see when this has happened.

Other Views and Wizards

Several other views also exist. For example, statistics view shows the statistics (frequencies) of all the events in a trace and the properties view shows the details about the properties of a trace (e.g., size, location, etc.). New views can also be added by extending a Java interface. In addition, several wizards allows to parse text files using regular expressions, parse XML files, and create and tune anomaly detection models.
