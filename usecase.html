TotalADS Use Case

This use case discusses the experiments we performed on a Linux server using TotalADS.

We used an Ubuntu Linux server for our use case. We equipped the server with Ubuntu Linux 12.04 operating system with Apache 2.2.17 web server, PHP 5.3.5 server side scripting engine, TikiWiki 8.1 content management system, FTP server, MySQL distribution 5.1.56 (version 14.14) database management system and an SSH server. These versions of the application software systems were selected because of the well-known vulnerabilities. To exploit these vulnerabilities in these software applications, we used an open source penetration testing tool Metasploit Framework (MSF). We attacked the Linux server with Metasploit and monitored the server by using TotalADS.

The overall process of this use case is as follows. First, we used the Linux server as one would use normally by executing different tasks, such as web serving, ssh access, execution of SQL queries, word processing and FTP access. During the execution of these tasks, we used TotalADS to collect system call traces automatically from the server and train different anomaly detection algorithms, such as HMM, Sequence Matching and Kernel State Modeling, on the system calls. Second, we stopped training when the anomaly detection model became stable. Third, we continued the normal use of the server and started monitoring the server using TotalADS in the testing mode. Fourth, we started attacking the server from Metasploit and monitored anomalies from TotalADS

TotalADS raised alarms of anomalies during the execution of attacks, which we further diagnosed by investigating the individual traces in views provided by TotalADS. This use case is discussed in detail in the enclosed article: download here

TotalADs is not only useful in diagnosing security exploits but it can also diagnose performance failures on a system in a similar manner as described earlier. It can also be used with logs in XMl format and logs in text format.
